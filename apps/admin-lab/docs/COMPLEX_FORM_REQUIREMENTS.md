# 상품 등록 폼 요구사항 정의서

## 📋 문서 정보

- **프로젝트**: Admin Lab - 복잡한 폼 리팩토링 과제
- **대상**: B2C + B2B 혼합 상품 등록/수정 어드민 폼
- **목적**: 리팩토링 실습을 위한 기능 명세
- **버전**: 1.0.0
- **작성일**: 2026-01-26

---

## 🎯 프로젝트 개요

### 배경

전자상거래 플랫폼에서 다양한 유형의 상품을 등록하고 관리할 수 있는 통합 어드민 폼이 필요합니다. 단일 상품, 옵션형 상품, 구독형 상품을 하나의 폼에서 처리하며, 재고 관리, 노출 설정, 배송 정보까지 포함합니다.

### 목표

- 단일 페이지에서 상품의 모든 정보를 입력/수정할 수 있어야 함
- 상품 유형에 따라 동적으로 UI가 변경되어야 함
- 실시간 validation 및 사용자 피드백 제공
- 임시 저장 기능으로 작업 중단 시 데이터 보존

---

## 📦 기능 요구사항

### 1. 기본 정보 섹션

#### 1.1 상품명

- **타입**: 텍스트 입력
- **필수 여부**: 필수
- **제약 조건**:
  - 최소 2자, 최대 40자
  - 특수문자 사용 불가
  - 앞뒤 공백 제거
- **Validation**:
  - 실시간 글자 수 표시
  - 2자 미만: "상품명은 최소 2자 이상이어야 합니다"
  - 40자 초과: "상품명은 최대 40자까지 입력 가능합니다"
  - 특수문자 포함 시: "상품명에 특수문자는 사용할 수 없습니다"
- **부가 기능**:
  - 자동 완성 제안 (입력값 기반)
  - 마지막 유효 입력 표시

#### 1.2 상품 코드

- **타입**: 텍스트 입력
- **필수 여부**: 필수
- **제약 조건**:
  - 자동 생성 (PROD-{timestamp} 형식)
  - 수정 가능
  - 영문 대문자, 숫자, 하이픈(-), 언더스코어(\_)만 사용
  - 최소 5자, 최대 50자
  - 하이픈으로 시작/종료 불가
  - 연속된 하이픈 불가
- **Validation**:
  - 비동기 중복 검사 (1초 debounce)
  - 형식 검증 (실시간)
  - 중복 시: "이미 사용 중인 상품 코드입니다"
- **부가 기능**:
  - 중복 확인 횟수 표시
  - 자동 생성된 코드 표시

#### 1.3 상품 설명

- **타입**: 텍스트 영역 (textarea)
- **필수 여부**: 선택
- **제약 조건**:
  - 최대 500자
  - 공백만 입력 불가
- **Validation**:
  - 실시간 글자 수 표시
  - 500자 초과: "상품 설명은 500자 이하여야 합니다"
- **부가 기능**:
  - 설명 템플릿 제공
  - 길이별 권장 메시지 표시

#### 1.4 상품 유형

- **타입**: 라디오 버튼
- **필수 여부**: 필수
- **선택지**: 단일 / 옵션형 / 구독형
- **동작**:
  - 유형 변경 시 하위 섹션 UI 전체 변경
  - 변경 시 확인 메시지 표시 (데이터 손실 경고)
- **부가 기능**:
  - 각 유형별 설명 표시
  - 유형 변경 히스토리 추적

### 2. 가격 정책 섹션

상품 유형에 따라 다른 UI를 표시합니다.

#### 2.1 단일 상품 가격

##### 2.1.1 판매가

- **타입**: 숫자 입력
- **필수 여부**: 필수
- **제약 조건**:
  - 0보다 커야 함
  - 최대 10,000,000원
  - 10원 단위로 입력
- **Validation**:
  - "판매가는 0보다 커야 합니다"
  - "판매가는 1천만원을 초과할 수 없습니다"
  - "판매가는 10원 단위로 입력해주세요"
- **부가 기능**:
  - 가격 변경 이력 표시
  - 시장 가격 비교 (가상 데이터)
  - 추천 가격 제공

##### 2.1.2 할인율

- **타입**: 숫자 입력 (%)
- **필수 여부**: 선택
- **제약 조건**:
  - 0~100%
  - 70% 초과 시 경고
  - 50% 초과 시 주의 메시지
- **Validation**:
  - "할인율은 0 이상이어야 합니다"
  - "할인율은 100% 이하여야 합니다"
- **부가 기능**:
  - 할인 금액 자동 계산 표시

##### 2.1.3 최종가

- **타입**: 읽기 전용 (자동 계산)
- **계산식**: `판매가 * (1 - 할인율/100)`
- **표시 정보**:
  - 원가 대비 퍼센트
  - 절약 금액
  - 고객 혜택 요약

#### 2.2 옵션형 상품

##### 2.2.1 옵션 그룹

- **기능**: 동적 추가/삭제/복제/순서 변경
- **구성**:
  - 옵션 그룹명 (필수)
  - 선택 방식: 단일 선택 / 다중 선택
- **제약**:
  - 최소 1개 그룹 필요
  - 그룹명 중복 허용

##### 2.2.2 옵션 아이템

- **기능**: 그룹 내 동적 추가/삭제
- **구성**:
  - 옵션명 (필수)
  - 추가 금액 (양수/음수 가능)
  - 재고 수량
- **제약**:
  - 그룹당 최소 1개 아이템 권장
  - 옵션명 중복 시 경고

##### 2.2.3 가격 통계

- **자동 계산 항목**:
  - 최저 가격 (minPrice)
  - 최고 가격 (maxPrice)
  - 평균 가격
  - 가격 범위
- **재고 통계**:
  - 전체 옵션 수
  - 판매 가능 옵션
  - 품절 옵션
  - 재고 부족 옵션 (10개 미만)

##### 2.2.4 Validation

- **경고 조건**:
  - 옵션 간 가격 차이가 100,000원 초과
  - 음수 추가 금액 옵션 존재
  - 옵션명 중복
- **제약**:
  - 옵션이 1개라도 있으면 단일 가격 필드 비활성화

#### 2.3 구독형 상품

##### 2.3.1 결제 주기

- **타입**: 라디오 버튼
- **필수 여부**: 필수
- **선택지**: 주 / 월 / 년
- **동작**: 주기 변경 시 입력 단위 자동 변경

##### 2.3.2 최소 구독 기간

- **타입**: 숫자 입력
- **필수 여부**: 필수
- **제약**:
  - 최소 1 (주/월/년)
  - 12개월 초과 시 경고 (고객 부담 증가)
- **부가 기능**:
  - 개월 수로 환산 표시
  - 총 구독 기간 표시

##### 2.3.3 해지 위약금

- **타입**: 체크박스 + 숫자 입력
- **필수 여부**: 선택
- **제약**:
  - 상품 가격의 50% 초과 시 법적 검토 필요 경고
- **부가 기능**:
  - 상품 가격 대비 비율 표시
  - 위약금 적용 조건 안내

##### 2.3.4 구독 수익 계산기

- **표시 항목**:
  - 총 구독 금액
  - 월 평균 금액
  - 예상 실제 매출 (유지율 70% 가정)
- **목적**: 사업성 판단 지원

### 3. 재고 / 판매 조건 섹션

#### 3.1 재고 관리 사용 여부

- **타입**: 토글 (체크박스)
- **기본값**: 비활성
- **동작**: OFF 시 재고 관련 필드 숨김

#### 3.2 재고 수량 (단일/구독형)

- **타입**: 숫자 입력
- **필수 여부**: 재고 관리 사용 시 필수
- **제약**:
  - 음수 불가
  - 최대 1,000,000
- **부가 기능**:
  - +10, +100 빠른 증가 버튼
  - 재고 레벨 표시 (안전/부족/긴급/품절)
  - 예상 품절일 계산 (평균 일 판매량 기반)

#### 3.3 품절 처리 방식

- **타입**: 라디오 버튼
- **선택지**: 자동 / 수동
- **설명**:
  - 자동: 재고 0 시 자동 품절 상태 전환
  - 수동: 재고 0이어도 수동 설정 필요

#### 3.4 고급 재고 설정

##### 3.4.1 재고 부족 기준

- **타입**: 숫자 입력
- **기본값**: 10개
- **설명**: 이 수량 이하 시 경고 표시

##### 3.4.2 품절 이메일 알림

- **타입**: 체크박스
- **기능**: 품절 시 담당자에게 이메일 발송

##### 3.4.3 자동 재발주

- **타입**: 체크박스
- **하위 필드**:
  - 재발주 시점 (개수)
  - 재발주 수량 (개수)
- **로직**: 재고가 재발주 시점 이하로 떨어지면 자동 발주

##### 3.4.4 평균 일 판매량

- **타입**: 숫자 입력
- **용도**: 예상 품절일 계산

#### 3.5 재고 히스토리

- **기능**: 재고 변경 이력 추적
- **표시 정보**:
  - 변경 일시
  - 변경 후 수량
  - 변경 사유

#### 3.6 옵션형 상품 특수 처리

- **제약**: 옵션형 상품은 옵션 단위로만 재고 관리
- **안내**: 각 옵션 아이템의 재고를 가격 정책 섹션에서 입력하도록 안내

### 4. 노출 설정 섹션

#### 4.1 노출 기간

##### 4.1.1 상시 노출

- **타입**: 체크박스
- **동작**: 체크 시 날짜 입력 필드 숨김

##### 4.1.2 시작 일시

- **타입**: datetime-local
- **제약**:
  - 현재 시각 이후여야 함
  - 종료일보다 이전이어야 함
- **Validation**:
  - "시작일은 현재 시각 이후여야 합니다"
  - "시작일은 종료일 이전이어야 합니다"

##### 4.1.3 종료 일시

- **타입**: datetime-local
- **제약**:
  - 시작일보다 이후여야 함
- **Validation**:
  - "종료일은 시작일 이후여야 합니다"

##### 4.1.4 빠른 설정 버튼

- **제공 옵션**: 1주일 / 1개월 / 3개월 / 1년
- **기능**: 내일부터 해당 기간 자동 설정

##### 4.1.5 타임존

- **타입**: 선택 (select)
- **선택지**: Asia/Seoul, America/New_York, Europe/London, Asia/Tokyo
- **안내**: 서버는 UTC 기준 저장

#### 4.2 노출 채널

##### 4.2.1 웹 (PC)

- **타입**: 체크박스
- **정보**: 예상 도달 일 10,000명

##### 4.2.2 모바일 (앱/웹)

- **타입**: 체크박스
- **정보**: 예상 도달 일 15,000명

##### 4.2.3 외부 마켓

- **타입**: 체크박스
- **동작**: 체크 시 연동 팝업 자동 실행
- **정보**: 예상 도달 일 20,000명

#### 4.3 외부 마켓 연동

##### 4.3.1 연동 프로세스

1. "외부 마켓" 체크박스 선택
2. 마켓 계정 선택 팝업 표시
3. 계정 선택 및 인증
4. 연동 완료 시 마켓 ID 반환

##### 4.3.2 연동 상태 관리

- **이미 연동된 경우**: 재연동 불가 안내
- **연동 중 submit 시도**: 차단 및 안내
- **연동 취소**: 체크박스 상태 rollback

##### 4.3.3 연동 정보 표시

- 마켓 플랫폼명 (네이버/쿠팡/G마켓)
- 마켓 ID
- 수수료율 (5~10%)
- 재고/가격 자동 동기화 옵션

#### 4.4 예상 도달 범위

- **계산**: 선택한 채널들의 예상 도달 수 합계
- **표시**: 일 도달 인원수
- **피드백**: 30,000명 초과 시 "높은 노출 효과 예상" 메시지

#### 4.5 Validation

- **최소 채널 선택**: 1개 이상 채널 선택 필요
- **경고**: 채널 미선택 시 경고 메시지 표시

### 5. 배송 정보 섹션

#### 5.1 구독형 상품 예외 처리

- **조건**: 상품 유형이 "구독형"인 경우
- **동작**: 배송 섹션 전체 비활성화
- **안내**: "구독형 상품은 배송 설정을 사용하지 않습니다"

#### 5.2 배송 유형

##### 5.2.1 무료 배송

- **설명**: 모든 주문에 무료 배송 제공
- **추가 필드**: 없음

##### 5.2.2 유료 배송

- **설명**: 모든 주문에 배송비 부과
- **추가 필드**:
  - 배송비 (필수)
  - 도서산간 추가비 (선택)

##### 5.2.3 조건부 배송

- **설명**: 구매 금액 조건 충족 시 무료
- **추가 필드**:
  - 배송비 (필수)
  - 도서산간 추가비 (선택)
  - 무료 배송 조건 금액 (필수)

#### 5.3 배송비

- **타입**: 숫자 입력
- **제약**: 0 이상
- **빠른 설정**: 2,500원 / 3,000원 / 5,000원 버튼

#### 5.4 도서산간 추가비

- **타입**: 숫자 입력
- **설명**: 제주도 및 도서산간 지역 배송 시 추가 비용

#### 5.5 무료 배송 조건 금액

- **타입**: 숫자 입력
- **필수 여부**: 조건부 배송 시 필수
- **제약**: 최소 10,000원 이상
- **Validation**: "무료 배송 조건은 최소 10,000원 이상이어야 합니다"
- **빠른 설정**: 30,000원 / 50,000원 / 100,000원 버튼

#### 5.6 배송 상세 설정

##### 5.6.1 배송 회사

- **타입**: 선택 (select)
- **선택지**: CJ대한통운, 우체국택배, 롯데택배, 한진택배, DHL, FedEx, 쿠팡로켓배송 등

##### 5.6.2 예상 배송 기간

- **타입**: 숫자 입력 (일 단위)
- **기본값**: 3일
- **설명**: 고객에게 안내되는 예상 소요 시간

##### 5.6.3 당일배송 옵션

- **타입**: 체크박스
- **추가 필드**:
  - 당일배송 추가 비용
- **설명**: 오후 2시 이전 주문 시 당일 배송 가능

##### 5.6.4 예약배송

- **타입**: 체크박스
- **설명**: 날짜/시간 지정 배송 가능

##### 5.6.5 포장 크기

- **타입**: 라디오 버튼

## 🔍 Validation 요구사항

### 검증 레벨

#### Level 1: 필드 단위 (실시간)

- **트리거**: onChange, onBlur
- **대상**: 개별 필드 규칙
- **표시**: 해당 필드 하단에 에러 메시지

#### Level 2: 섹션 단위

- **트리거**: 섹션 완료 시 or 다음 단계 이동 시
- **대상**: 섹션 내 모든 필드
- **표시**: 섹션 상단에 요약 메시지

#### Level 3: 전체 폼 (제출 시)

- **트리거**: Submit 버튼 클릭
- **대상**: 모든 필드 + 섹션 간 의존성
- **표시**: 페이지 상단 에러 요약

### 검증 규칙 분류

#### 필수 검증 (Critical)

- 상품명, 상품 코드, 상품 유형
- 단일 상품: 판매가
- 옵션형 상품: 최소 1개 옵션 그룹
- 조건부 배송: 무료 배송 조건 금액

#### 경고 (Warning)

- 할인율 50% 초과
- 할인율 70% 초과 (승인 필요)
- 12개월 초과 구독 기간
- 옵션 가격 차이 100,000원 초과
- 재고 부족 상태

### 검증 타이밍

#### 즉시 표시 (Immediate)

- 글자 수 제한 (상품명, 설명)
- 형식 검증 (상품 코드)
- 숫자 범위 (가격, 재고)

#### Submit 시 표시 (On Submit)

- 필수 필드 누락
- 섹션 간 의존성 (예: 조건부 배송 금액)
- 최종 비즈니스 규칙

#### 다른 필드 수정 시 사라짐 (Context-dependent)

- 날짜 검증 (시작일 ↔ 종료일)
- 옵션 수량 변경 시 가격 범위 재계산

## 🚨 제약 사항 및 특수 규칙

### 상품 유형별 제약

#### 단일 상품

#### 구독형 상품

- 배송 섹션 비활성화
- 결제 주기 필수
- 최소 구독 기간 필수
- 해지 위약금 선택

### 필드 간 의존성

#### 가격 계산

- 최종가 = 판매가 × (1 - 할인율/100)
- 자동 계산 후 표시 (실시간)

#### 옵션 가격 범위

- 최저가 = 모든 옵션 중 최소 추가 금액
- 최고가 = 모든 옵션 중 최대 추가 금액
- 평균가 = 모든 옵션 추가 금액의 평균
- 자동 계산 후 표시

#### 구독 금액 계산

- 월 환산 = 최소 구독 기간 × 주기 배수
- 총 구독 금액 = 기본 가격 × 월 환산
- 예상 매출 = 총 구독 금액 × 0.7 (유지율)

#### 재고 경고

- 안전: 재고 > 재고부족기준
- 부족: 재발주시점 < 재고 ≤ 재고부족기준
- 긴급: 0 < 재고 ≤ 재발주시점
- 품절: 재고 = 0

#### 노출 채널 예상 도달

- 웹: +10,000
- 모바일: +15,000
- 외부마켓: +20,000
- 총합 계산 후 표시

### 외부 연동 제약

#### 마켓 연동

- 연동 중에는 Submit 불가
- 이미 연동된 경우 재연동 불가
- 연동 취소 시 체크박스 rollback

#### 상품 코드 중복 체크

- 1초 debounce
- 비동기 API 호출
- 결과 표시 (사용 가능 / 중복 / 형식 오류)

interface FormState {
  // 기본 정보
  productName: string; // 2~40자
  productCode: string; // 5~50자, 영문대문자/숫자/하이픈/언더스코어

  salePrice: number; // > 0

  optionGroups: OptionGroup[]; // 동적 배열
  minPrice: number; // 자동 계산
  maxPrice: number; // 자동 계산

  // 가격 - 구독형
  paymentCycle: '주' | '월' | '년';
  minSubscriptionPeriod: number; // >= 1
  hasCancellationFee: boolean;
  cancellationFee: number; // >= 0

  // 재고
  useStockManagement: boolean;
  stockQuantity: number; // >= 0
  stockType: '자동' | '수동';

  // 노출 설정
  startDate: string; // ISO 8601
  endDate: string; // ISO 8601
  showWeb: boolean;
  showMobile: boolean;
  showExternalMarket: boolean;
  externalMarketId: string;

  // 배송
  deliveryType: '무료' | '유료' | '조건부';
  deliveryFee: number; // >= 0
  remoteAreaFee: number; // >= 0
  conditionalAmount: number; // >= 10000 (조건부 시)

  // 메타
  errors: Record<string, string>;
  touched: Record<string, boolean>;
  isDraft: boolean;
  isSubmitting: boolean;
}
